# 🌍 Origem do Git

## 📖 Quando e por que o Git foi criado?

O **Git** foi criado em **2005** por **Linus Torvalds**, o mesmo criador do kernel Linux. Seu desenvolvimento surgiu durante uma crise no processo de versionamento do próprio projeto Linux.

![Linus Torvalds](https://upload.wikimedia.org/wikipedia/commons/6/69/Linus_Torvalds.jpeg)

Na época, os desenvolvedores utilizavam uma ferramenta chamada **BitKeeper**, um sistema de controle de versão proprietário. Quando a licença gratuita deixou de ser disponibilizada para o desenvolvimento do Linux, tornou-se necessário criar uma alternativa própria, eficiente e confiável.

Diante desse cenário, Linus Torvalds decidiu desenvolver uma nova ferramenta de controle de versão que atendesse às necessidades do projeto Linux.

---

## 🕰️ Contexto histórico

- O kernel Linux era mantido por milhares de desenvolvedores ao redor do mundo.
- O projeto dependia do BitKeeper para controlar alterações no código.
- Em 2005, mudanças na licença da ferramenta impediram seu uso gratuito pela comunidade Linux.
- Como solução, Linus Torvalds iniciou o desenvolvimento do Git.
- A primeira versão funcional do Git foi criada em apenas **10 dias**.

Durante o desenvolvimento, Linus Torvalds afirmou:

> “Eu queria algo que fosse realmente rápido.”

Essa preocupação com desempenho acabou se tornando uma das principais características do Git.

---

## 🎯 Objetivos do Git

O Git foi projetado com foco em três características principais:

### ⚡ Velocidade

Executar operações de versionamento de forma extremamente rápida, mesmo em projetos gigantescos.

### 🌐 Sistema Distribuído

Cada desenvolvedor possui uma cópia completa do repositório, incluindo todo o histórico de versões.

### 🔒 Segurança e Confiabilidade

O Git utiliza mecanismos de integridade que ajudam a proteger o histórico do projeto contra corrupção ou perda de dados.

---

## 💡 Por que o Git se tornou tão importante?

O Git revolucionou o desenvolvimento de software porque facilitou:

- O trabalho colaborativo entre equipes
- O controle completo das alterações do projeto
- A criação de versões seguras do código
- O desenvolvimento simultâneo de funcionalidades
- A recuperação de versões anteriores do sistema

Hoje, o Git é o sistema de controle de versão mais utilizado no mundo.

---

## 🧠 Curiosidades sobre o Git

- O nome **Git** foi escolhido por Linus Torvalds de forma bem-humorada. Em inglês britânico, a palavra pode significar algo como “pessoa teimosa” ou “idiota”.
- O Git foi criado inicialmente para atender às necessidades do desenvolvimento do kernel Linux.
- Mesmo após tantos anos, o Git continua sendo a principal ferramenta de versionamento utilizada por empresas e desenvolvedores no mundo inteiro.
- Plataformas gigantes como GitHub, GitLab e Bitbucket são baseadas no Git.
